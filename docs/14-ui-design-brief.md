# UI Design Brief — Bartar CRM (فاز ۱)

این متن برای طراحی رابط کاربری (مثلاً در Claude Design) نوشته شده.
خروجی طراحی باید قابل پیاده‌سازی با Next.js 14 + Tailwind + shadcn/ui باشد.

---

## محصول
وب‌اپ داخلی مدیریت تعمیرات و خرید قطعات برای یک مرکز تعمیرات تجهیزات الکترونیکی.
جایگزین برگه‌های کاغذی خرید قطعه. کاربران: پذیرش، تعمیرکار، خریدار، اپراتور قیمت، ادمین، مدیر.

## اصول (غیرقابل مذاکره)
- **RTL و کاملاً فارسی** — فونت پیشنهادی: Vazirmatn
- **Mobile First** — اکثر کاربران (تعمیرکار و خریدار) با موبایل کار می‌کنند. طراحی از عرض 380px شروع شود، بعد دسکتاپ.
- حداقل کلیک، فرم‌های ساده، دکمه‌های بزرگ (حداقل ارتفاع لمسی 44px)
- رنگ‌بندی وضعیت‌ها در کل سیستم یکسان باشد
- Component-Based و سازگار با توکن‌های shadcn/ui (primary, muted, destructive, ...)

## ناوبری
- موبایل: Bottom Navigation با ۴ آیتم: داشبورد، درخواست‌ها، خرید، بیشتر (منوی کشویی: کاربران، نقش‌ها، فروشندگان، قیمت‌ها، لاگ‌ها، خروج)
- دسکتاپ: Sidebar راست‌چین
- آیتم‌های منو بر اساس دسترسی (Permission) کاربر نمایش داده می‌شوند

## رنگ وضعیت‌های درخواست قطعه (۱۳ وضعیت)
- خاکستری: ثبت شده (CREATED)
- زرد: در انتظار تایید مشتری (WAITING_CUSTOMER)
- آبی: در صف خرید (WAITING_PURCHASE)، در حال خرید (PURCHASING)
- سبز: تایید شد (APPROVED)، خرید شد (PURCHASED)، تحویل شد (DELIVERED)، مصرف شد (CONSUMED)
- قرمز: رد شد (REJECTED)، لغو شد (CANCELLED)، عدم موجودی (NOT_FOUND)
- نارنجی: مرجوع شد (RETURNED)
- تیره/بسته: بسته شد (CLOSED)

## صفحات مورد نیاز

### 1. ورود (موجود — برای یکدستی بازطراحی شود)
شماره موبایل + رمز عبور + دکمه ورود + نمایش خطا

### 2. داشبورد
کارت‌های آمار (درخواست‌های باز، در صف خرید، خریدهای امروز، عدم موجودی‌ها) + لیست ۵ درخواست آخر با Badge وضعیت

### 3. کاربران (ادمین)
- لیست: موبایل = کارت (نام، شماره، Badgeهای نقش، وضعیت فعال/غیرفعال)؛ دسکتاپ = جدول
- جستجو + فیلتر فعال/غیرفعال + دکمه «کاربر جدید»
- فرم ساخت/ویرایش: نام، موبایل، ایمیل (اختیاری)، رمز عبور، انتخاب چندتایی نقش، سوییچ فعال
- تایید دو مرحله‌ای برای حذف

### 4. نقش‌ها و دسترسی‌ها (ادمین)
- لیست نقش‌ها: نام، توضیح، تعداد کاربر، تعداد دسترسی
- فرم نقش: نام + توضیح + **ماتریس دسترسی‌ها**: چک‌باکس‌ها گروه‌بندی‌شده بر اساس group
  (Authentication, Repair, Part Request, Purchase, Pricing, Vendor, Users, Roles, Logs, Reports, Settings)
  با قابلیت «انتخاب همه‌ی گروه»

### 5. پذیرش / تیکت تعمیر
- فرم ثبت: انتخاب/ساخت سریع مشتری (نام + موبایل)، برند، مدل، سریال، شرح ایراد
- لیست تیکت‌ها با شماره پذیرش و وضعیت

### 6. درخواست قطعه (قلب سیستم)
- فرم ثبت: انتخاب تیکت تعمیر، نام قطعه، کیفیت (اورجینال/های‌کپی/کپی)، تعداد، توضیح
- لیست با فیلتر وضعیت (Chipهای رنگی) + جستجو
- **صفحه جزئیات**: اطلاعات کامل + تایم‌لاین عمودی تاریخچه‌ی وضعیت (چه‌کسی، کی، چه وضعیتی) + دکمه‌های اکشن متناسب با وضعیت فعلی (اعلام هزینه، تایید مشتری، شروع خرید، ...)

### 7. صف خرید (خریدار — مهم‌ترین صفحه‌ی موبایل)
- لیست کارت‌های درخواست در صف با اطلاعات کلیدی
- فرم ثبت خرید: فروشنده (انتخاب/ساخت سریع)، قیمت خرید، تعداد
- **هشدار اختلاف قیمت**: اگر قیمت خرید با قیمت اعلامی اختلاف داشت Alert زرد نمایش داده شود ولی ثبت ممکن باشد
- دکمه‌های «عدم موجودی» و «مرجوعی»

### 8. فروشندگان
لیست + فرم (نام، موبایل، آدرس، توضیح)

### 9. قیمت‌ها
لیست قطعات با آخرین قیمت خرید/فروش + مودال تاریخچه‌ی قیمت (نمودار ساده یا جدول)

### 10. لاگ فعالیت‌ها (ادمین/مدیر)
لیست با فیلتر کاربر/نوع عملیات/بازه تاریخ — هر آیتم: کاربر، عملیات، موجودیت، زمان

## حالت‌های عمومی هر صفحه
Loading (Skeleton) / Empty State (پیام + آیکون + CTA) / Error State / تایید عملیات مخرب با Dialog

## خروجی مطلوب
صفحات کامل موبایل + دسکتاپ برای هر مورد بالا، با کامپوننت‌های تفکیک‌شده (Card, Badge, StatusChip, Timeline, PermissionMatrix, BottomNav, ...)
